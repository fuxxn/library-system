# 圖書借閱者分類與推薦系統

以 Node.js + Express 建立的全端 Web 應用程式，透過多層決策樹將借閱者分成四種類型，並依據分類給出個人化推薦書單。決策樹的門檻值由 `compute_ig.js` 對實際借閱資料計算**資訊增益（Information Gain）**自動決定，而非手動設定。

## 環境需求

- Node.js **v22.5.0 以上**（使用內建 `node:sqlite`，不需額外安裝資料庫驅動）

## 啟動方式

```bash
npm install
node index.js
```

瀏覽器開啟 → http://localhost:3000

首次啟動會自動建立 `library.db` 並寫入種子資料（50 本書、20 位借閱者、~255 筆紀錄）。

## 功能說明

### 1. 書籍管理

- **瀏覽書籍**：以表格呈現全部書籍（書名、類型、作者），支援依類型篩選（小說、科普、漫畫、商業、歷史）
- **新增書籍**：填寫書名、類型、作者後送出，立即出現在列表
- **修改書籍**：點選書籍右側「編輯」按鈕，在原位修改書名、類型或作者
- **刪除書籍**：點選「刪除」按鈕，連同該書所有借閱紀錄一併移除

### 2. 借閱操作

- 從下拉選單選擇書籍、輸入借閱者姓名、選取借閱日期後送出
- 成功後顯示提示訊息，借閱者名單即時更新

### 3. 個人化推薦

輸入或選擇借閱者姓名後，系統會：

1. 計算該借閱者的兩個特徵值：
   - **總借閱次數**（total_borrows）
   - **偏好集中度**（top_genre_ratio）：最常借類型的次數 ÷ 總借閱次數
2. 透過決策樹判斷借閱者類型（見下方說明）
3. 顯示帶顏色標籤的分類結果，以及對應的推薦書單

| 借閱者類型 | 推薦策略 |
|-----------|---------|
| 🔴 專注型重度讀者 | 同類型中最多人借閱的熱門書 |
| 🟠 博覽型重度讀者 | 每種類型各推薦一本精選書 |
| 🔵 專注型輕度讀者 | 同類型中借閱次數最少的入門書 |
| 🟢 探索型輕度讀者 | 全站借閱次數最多的熱門書 |

---

## 決策樹分類邏輯

位於 `decisionTree.js`，採用兩層巢狀結構：

```
total_borrows > 11.5  →  重度讀者（IG = 0.9984）
  ├─ top_genre_ratio > 0.5167  →  專注型重度讀者
  └─ top_genre_ratio ≤ 0.5167  →  博覽型重度讀者

total_borrows ≤ 11.5  →  輕度讀者
  ├─ top_genre_ratio > 0.7083  →  專注型輕度讀者（IG = 1.0000）
  └─ top_genre_ratio ≤ 0.7083  →  探索型輕度讀者（IG = 1.0000）
```

### 資訊增益計算（進階功能）

執行 `node compute_ig.js`，程式會從資料庫讀取實際資料，對以下 4 個候選特徵掃描所有可能的切割點，找出使資訊增益最大的特徵與門檻值：

| 候選特徵 | 說明 |
|---------|------|
| `total_borrows` | 總借閱次數 |
| `top_genre_ratio` | 偏好集中度（最多類型次數 ÷ 總借閱次數） |
| `distinct_books` | 借過的不同書籍數 |
| `avg_interval` | 平均借閱間隔天數 |

計算結果：
- 輸出各層最佳特徵與門檻的 IG 條狀圖
- 用找到的門檻重新分類 20 位種子借閱者，驗證準確率（100%）
- 結果寫入 `ig_result.json`

## API 端點

| 方法 | 路徑 | 說明 |
|------|------|------|
| GET | /api/books | 取得所有書籍，支援 `?genre=小說` 篩選 |
| POST | /api/books | 新增書籍 `{ title, genre, author }` |
| PUT | /api/books/:id | 修改書籍 |
| DELETE | /api/books/:id | 刪除書籍（連同借閱紀錄） |
| POST | /api/borrow | 新增借閱紀錄 `{ book_id, borrower_name, borrow_date }` |
| GET | /api/borrowers | 取得所有不重複的借閱者名稱 |
| GET | /api/borrowers/:name/type | 回傳分類結果與特徵數值 |
| GET | /api/borrowers/:name/recommendations | 依分類回傳推薦書單（3–5 本） |

## 專案結構

```
library-system/
├── index.js          # Express 主程式、所有 API 路由
├── decisionTree.js   # 決策樹分類邏輯（IG 自動計算門檻）
├── seed.js           # 種子資料（50 本書、20 位借閱者、~255 筆紀錄）
├── compute_ig.js     # 資訊增益計算腳本（node compute_ig.js）
├── ig_result.json    # compute_ig.js 的輸出結果
├── package.json
└── public/
    └── index.html    # 前端單頁應用（Vanilla JS）
```

## 種子資料設計

| 類型 | 人數 | 借閱次數 | 偏好集中度 |
|------|------|---------|-----------|
| 專注型重度讀者 | 5 | 16–24 次 | ≥ 85%（同一類型） |
| 博覽型重度讀者 | 5 | 15–25 次 | ~20%（均衡分散） |
| 專注型輕度讀者 | 5 | 4–8 次 | ≥ 85%（同一類型） |
| 探索型輕度讀者 | 5 | 3–7 次 | ≤ 40%（類型分散） |
